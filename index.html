Tabula Rasa - Content Management System

Year: 2004

Ambition: Building a highly customizable, intuitive and scalable CRM-system. The system had to support several levels of security for users and groups tied to several languages and several markets. Administrators in one market can easily set up a whole site by copying data from another market and translate language, but still be able to have a completly different design. The system also allowed for external linking but with embedded search of that linked content. The back-end sported a TinyMCE-editor, drag-n-drop arranging of the meny-structure and ability to build drafts before publishing. The system was used by a large medical coorporation for all their subsidiaries with multi-language support.

Architecture: The front-end uses a custom-made template-system in HTML4.01 Strict and PHP5 as engine. This allows for different designs with same dataset. The back-end relies on PHP5 and the javascript framework prototype to enable animations and drag-n-drop support. The interface for the CMS was inspired by K.I.S.S. and the goal was to keep the interface clean.

Demo Available: Yes! Visit http://ortho.kanzie.com/ to see dummy-site and http://ortho.kanzie.com/admin.php to log in and make updates.

deepFLING Point of Sale-system

Year: 2008

Ambition: To build a customizable and agile point-of-sale system that is shaped to support the business by enforcing policies but allow for exceptions. The system guides and facilitates the user to a correct and efficient sale. The system automates sale-statistics, bookeeping and commision-calculation as well as improves order-handling and correct inventory with the webstore. Automatic price-suggestion and markup-tracking of individual items as well as a re-worked and normalized inventory-system was included in the release.

Architecture: PHP5 using OOP, jQuery framework for logic on client, MySQL 5 using a combination of InnoDB and MyISAM. Singleton database-layer and JSON for data between server and client. Client verifications of data consistency as well as server check and validation with exception-handling and transaction-support. Client performs a round-robin validity-check of what was store in database per checkout and boasts automatic logging of errors, inconsistencies and discrepancies to administrators without interrupting user.

Gaiaview - a map service for a better world

Year: 2006

Ambition: By using Google Maps as a base-layer and let different organizations highlight areas of interest and tie that to pictures, document and movies in real-time. I wanted to show how a crisis-area in the world is topic for several organizations in different ways but also let a map be the index on which information is stacked.

Architecture: After reviewing several techniques to solve our need to draw on top of the map before the API itself had this functionality we decided to move ahead with a combination of Flash and PHP5+js. Flash is used in the studio where maps are create and in the view-mode html4.01 and js was used. A logic-layer to handle all map-data and stream pointers from server to client as well as re-draw and bounding-box tests was written in javascript. This allowed the system to scale nicely by utilizing the power of each connected client instead of using the server.

Presentation folder: Download the now public presentation I wrote at http://cv.kanzie.com/docs/gaiaview.pdf.
